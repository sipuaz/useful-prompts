# Foundations

## What this section is

Before picking up any specific prompting technique, there are a few concepts that everything else builds on. This section covers those fundamentals.

Think of it as the difference between knowing how to use a hammer vs. understanding what nails are for. You can copy techniques without this layer — but you won't know when to use them, why they work, or how to debug them when they don't.

## Who this is for

Someone who:
- Is new to working with LLMs seriously (not just casual chatting)
- Has a technical background (dev, devops, data) and wants mental models, not just recipes
- Will go on to read the technique folders in this repo

## How to use this section

Read [`anatomy-of-a-prompt/README.md`](./anatomy-of-a-prompt/README.md) first. Then grab [`anatomy-of-a-prompt/template.md`](./anatomy-of-a-prompt/template.md) and keep it handy when writing new prompts.

Every technique folder in `01-prompting-techniques/` and beyond will reference this foundation, so it's worth understanding before going deeper.
